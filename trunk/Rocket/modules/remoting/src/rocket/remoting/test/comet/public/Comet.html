Numerous buttons below are available which provide the capability to test many of the comet features. Each step that occurs in the browser or server is logged.

Things being tested
  • Starting a comet session (by clicking on the Start button)
  • Starting a comet session with an invalid url (by clicking on the Start using bad url button)which should result in a Exception
  • Terminating a session.
    • Terminate from the client by clicking on Client terminate
    • Terminate from the server by clicking on Server terminate
    • Killing the embedded tomcat and checking that the client complained
  • Click on Compile/Browse and repeat the test with other browsers
  • The server drops the connection forcing the client to reconnect after approximately 3 messages, scan log for "Too many bytes written to client, server will drop connection"
  • The server also drops the connection when the connection is open for more than 5 seconds, scan log for "Connection open too long server will drop connection".
